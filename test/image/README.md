# 图片问题案例与参考回复

本目录提供航运/海图场景的图片问答案例，供 agent 在处理图片问题时参考回答结构、专业术语和排查思路。

## 文件配对规则

- 文件名中包含用户原始问题的图片为**输入问题图**。
- 同编号、以 `-参考回复.png` 结尾的图片为该问题的**参考回复图**。
- 例如：
  - `image01-这个在全球海图里是什么意思.png`：用户输入图片，问题为“这个在全球海图里是什么意思”。
  - `image01-参考回复.png`：该问题的参考回复。
- `image04` 的输入文件使用 `image-04` 前缀，仍与 `image04-参考回复.png` 配对；`Image05-...png.png.png` 为 `image05-参考回复.png` 的输入图。

## 案例摘要

| 编号 | 用户问题 | 参考结论 | 回答要点 |
| --- | --- | --- | --- |
| image01 | 全球海图中的红色圆形航标是什么意思？ | 安全水域标（Safe Water Mark），表示该处及周边水域可安全航行，并非危险物标志。 | 说明其海图含义、常见设置位置和实体外观；同时与危险物标、沉船等符号区分。 |
| image02 | 图上紫色波浪/锯齿线是什么意思？ | 更可能是航行管制区、特殊海域等的边界线；也可能是海底电缆/管道，须结合图层和线形确认。 | 不可仅凭颜色断言；提示用户在平台的“显示—区域”等图层中开关核实，并说明邻近灰色实线及数字通常为等深线/水深值。 |
| image03 | 图中的小圈圈是什么意思？ | 电子海图中的指定锚泊（锚地）范围，用于提示船舶可按规定抛锚或锚泊。 | 说明锚地的用途和限制，提醒结合海图图例、公告、港口规定及当前航行状态确认，不能把任意圆形符号都解释为锚地。 |
| image04 | 平均航速是否包含停航时段、进出港时间？ | 图示“平均航速”按船舶处于移动状态的航行总里程除以移动总时间计算。 | 靠泊、锚泊、等待、系泊、完全静止/漂航等停航时段不计入；进出港、狭水道、避让等仍在移动的低速航段计入。可进一步与“航次总平均速度”区分。 |
| image05 | 两艘船连续两天没有船位跟踪，但 AIS 工况正常，周边船舶正常，可能是什么问题？ | 优先怀疑卫星 AIS 覆盖/接收条件，其次排查船端天线、发射功率、设备与平台数据。 | 结合船舶位置、距岸和周边接收情况分层排查；平台侧核验原始报文、数据刷新和 MMSI/IMO/船名信息，船端核验 AIS 状态、天线馈线、供电与发射功率。 |

## Agent 使用指南

1. **先识别图片事实**：读取颜色、线型、图标、文字、位置、时间范围和界面字段；不清晰时明确说明无法确认的部分。
2. **按案例确定回答类型**：符号释义类说明“含义 + 用途 + 易混淆项”；指标类说明“口径 + 计算范围 + 排除项”；故障类按“平台/数据—船端设备—外部环境”分层排查。
3. **避免过度确定**：海图符号常受图层、比例尺、海区和平台配置影响。对于 image02 一类线状要素，应给出主要可能性并提供图层、图例或坐标核实方法，而不是只按颜色下结论。
4. **给出可执行动作**：优先列出用户可立即完成的检查，例如查看图例/图层、提供坐标和截图、核验时间范围、查询报文记录、检查 AIS 天线和供电。
5. **保持航海安全边界**：回答用于辅助理解和故障初判，不替代官方海图、航海通告、港口规定、船长判断或专业技术支持。涉及实际航行、锚泊和设备故障时，应建议以官方资料和现场核验为准。

## 推荐回答模板

```text
结论：
先直接说明最可能的含义/判断，并标注是否需要进一步核实。

依据：
- 指出图片中的关键视觉或数据线索。
- 说明相关平台口径、海图图例或设备工作原理。

易混淆项：
- 列出 1～2 个相近概念及区别；无法仅凭图片确认时明确说明。

下一步：
- 给出用户可操作的核实或排查步骤。
```

## 使用限制

- 参考回复用于学习表达和排查框架，不应逐字照抄。
- 若图片内容、产品版本或业务规则与案例不同，应以当前图片、平台图例和最新业务资料为准。
- 输出时不要虚构已查询到的后台记录、AIS 报文、坐标或图层状态；没有数据时应说明所需的补充信息。
