- **分层架构**：采用 `schemas` (Pydantic DTO) -> `models` (SQLAlchemy ORM) -> `repository` (Raw SQL/CTE) -> `writer` (Async/Sync Facade) 的分层设计。
- **数据持久化**：底层依赖 PostgreSQL，通过 `sql/*.sql` 脚本在 `observability` Schema 下维护 `api_calls`、`tool_invocations`、`agent_errors` 及 `chat_debug_sessions` 表。
- **写入策略**：`writer.py` 提供 `schedule_*` 系列函数，利用 `ThreadPoolExecutor` 实现非阻塞的异步日志写入；同时提供 `log_*` 协程供 `asyncio` 环境直接调用。
- **查询优化**：`repository.py` 大量使用原生 SQL 和 CTE（公用表表达式）处理复杂的聚合统计（如仪表盘 KPI、趋势分析）及会话级关联查询，以规避 ORM 在复杂报表场景下的性能瓶颈。
- **Schema 自管理**：模块内置 `ensure_schema` 机制，在每次写操作前自动检查并执行增量 SQL 脚本，确保数据库结构与应用代码同步。
