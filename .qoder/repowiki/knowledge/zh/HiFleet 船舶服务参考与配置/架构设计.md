- 参考层 (`references/`)：以 Markdown 形式定义 API 契约，`api_base.md` 确立双域名路由（`api.hifleet.com` 用于查询，`ttseapi.hifleet.com` 用于搜索/上传）及三类 Key (`api_key`, `hifleet_key1`, `hifleet_key2`) 的鉴权边界。
- 配置层 (`config/`)：通过 `strait_oid_map.json` 维护咽喉航道 OID 与海域 ID 的中英双语映射，解耦业务语义与 API 参数。
- 依赖方向：业务逻辑依赖参考文档中的 URL 模板与参数规则，并读取配置文件完成请求参数的动态组装。
