# 经纬度格式转换规范

## 支持的输入格式

用户输入的经纬度可能有多种格式，需统一转换为十进制度（Decimal Degrees）才能上传到 API。

| 格式 | 示例 | 转换结果 |
|------|------|---------|
| 度分秒 | 116°52′30″E | 116.875° |
| 度分 | 116°52.5′E | 116.875° |
| 十进制度 | 116.875°E | 116.875° |
| 纯数字 | 116.875 | 116.875 |

## 转换规则

### 公式

```
十进制度 = 度 + 分/60 + 秒/3600
```

### 方向处理

| 方向 | 纬度 | 经度 |
|------|------|------|
| N（北纬） | 正数 | - |
| S（南纬） | 负数 | - |
| E（东经） | - | 正数 |
| W（西经） | - | 负数 |

## 转换示例

### 度分格式

```
输入：116°52.5′E
计算：116 + 52.5/60 = 116 + 0.875 = 116.875

输入：22°7.5′N
计算：22 + 7.5/60 = 22 + 0.125 = 22.125

输入：72°3.763′W
计算：-(72 + 3.763/60) = -(72 + 0.062717) = -72.062717
```

### 度分秒格式

```
输入：116°52′30″E
计算：116 + 52/60 + 30/3600 = 116 + 0.8667 + 0.0083 = 116.875

输入：22°7′30″N
计算：22 + 7/60 + 30/3600 = 22 + 0.1167 + 0.0083 = 22.125
```

## 常见错误

| 错误 | 正确 | 说明 |
|------|------|------|
| 116°52.5′ = 116.525 | 116°52.5′ = 116.875 | 52.5/60 ≠ 0.525 |
| 22°7.5′ = 22.75 | 22°7.5′ = 22.125 | 7.5/60 ≠ 0.75 |
| 72°3.763′ = 72.3763 | 72°3.763′ = 72.062717 | 3.763/60 ≠ 0.3763 |

## 正则提取

### 度分秒格式

```python
import re

# 度分秒：116°52′30″E 或 116°52'30"E
pattern = r"(\d+)[°]\s*(\d+)[′']\s*(\d+(?:\.\d+)?)[″\"]\s*([EWNSewns])"
```

### 度分格式

```python
# 度分：116°52.5′E 或 116°52.5'E
pattern = r"(\d+)[°]\s*(\d+(?:\.\d+)?)[′']\s*([EWNSewns])"
```
