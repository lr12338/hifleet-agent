# Skill: knowledge-qa

## 概述
知识检索与问答域，处理所有信息查询请求。统一使用 `smart_search`，内部自动 5 层路由，并优先使用结构化联网搜索结果而不是生成式搜索文本。

## 适用场景
- Hifleet平台功能咨询（如何注册、怎么用DTU、船位问题等）
- 航运行业知识（AIS原理、海事法规、航运术语等）
- 实时行业动态（运价、法规更新、市场趋势等）
- 产品功能介绍（视频监控、岸基值班、性能跟踪等）

## 工具

### smart_search
**统一信息检索入口**，根据depth参数自动选择搜索策略：

| depth | 搜索范围 | 适用场景 | 典型耗时 |
|-------|---------|---------|---------|
| quick | 术语速查→知识库FAQ→知识库Wiki | 平台概念、术语、简单FAQ | <3秒 |
| normal | 同quick + HiFleet 官网站内搜索；必要时补一轮公开网页搜索 | 一般航运/海事知识 | 5-15秒 |
| deep | 同normal + 多查询词结构化深搜（`web_summary`） | 时效敏感、深度分析、行业动态 | 15-60秒 |

## 联网搜索规则

`smart_search` 的联网层遵循以下约束：

1. **官方优先**
   - HiFleet 相关问题优先搜 `hifleet.com` / `help.hifleet.com`
   - 能用官方资料回答时，不依赖公开网页兜底

2. **结构化结果优先**
   - 优先读取搜索结果中的 `Summary`，不要把 `Snippet` 当主要依据
   - `Snippet` 只适合列表展示，正文/总结场景优先 `Summary`
   - 深搜时优先使用 `web_summary`，获取结构化结果和总结

3. **改写与深搜**
   - `quick` 默认不开启 QueryRewrite
   - `normal`/`deep` 在站内搜索或复杂公开搜索时可开启 QueryRewrite
   - `deep` 会对故障、异常、分析类问题生成少量查询变体并合并结果

4. **权威性判断**
   - 优先使用官方站点和高权威站点
   - 结合返回的 `AuthInfoLevel/AuthInfoDes` 做来源排序
   - 对低权威结果仅作补充，不作为唯一结论依据

5. **输出要求**
   - 返回给 agent 的搜索结果应简洁、结构化
   - 禁止输出 `AI摘要`、`回答指导`、`[Query1:...]` 这类模板噪音
   - 总结时优先保留：标题、摘要、发布时间、站点、可访问链接

**depth选择规则**：
1. 用户问"是什么/怎么用/如何操作" → depth="quick"
2. 用户问"最新/最近/今天/今年/行业动态" → depth="normal"
3. 用户需要"分析/趋势/深度/全面/对比" → depth="deep"
4. 不确定时默认 depth="normal"

**示例调用**：
- smart_search(query="绿点是什么", depth="quick") → 术语直接命中
- smart_search(query="AIS信号不好", depth="normal") → 知识库+搜索
- smart_search(query="航运市场运价趋势", depth="deep") → 多轮深度搜索

## 搜索结果处理

### FAQ/标准回复匹配
当检索到 FAQ 条目时，严格按照标准回复内容回答，不添加额外信息。

### 主题说明
当仅匹配到主题说明时，可基于主题内容谨慎回答，但需注明是概述而非精确 FAQ。

### 知识检索无结果
"抱歉，我暂时没有找到关于【问题】的准确答案。建议您：
1. 尝试换一种问法
2. 联系人工客服（客服电话：400-963-6899，微信客服：hifleetkhzs）
3. 访问帮助中心 https://help.hifleet.com"
