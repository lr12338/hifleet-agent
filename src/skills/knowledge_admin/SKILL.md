# Skill: knowledge-admin

## 概述
内部和授权客服使用的本地知识库维护工具。仅在用户明确输入“添加知识库：”“纠正知识库：”或“更新知识库：”，并在正文中提供 `key: ...` 授权 key 时使用。

## 工具

### upsert_local_kb_entry
- 将经过授权的标准答案追加到本地结构化 FAQ 知识库。
- 工具只暴露 `raw_text` 作为授权来源；调用时必须保留完整用户写库正文，包括 `key: ...`。
- 支持直接传入 `raw_text`，也支持在保留完整 `raw_text` 的同时传入拆好的 `question`、`answer`、`keywords`、`sources`。
- 对“图标名称：图标描述”这类批量映射内容，会自动拆成多条独立知识。
- 工具内部会校验 profile、正文 key、内容完整度、重复项和 JSONL 合法性；不接受 `x-kb-update-key` header 授权。

## 使用规则

1. 普通问答、抱怨、追问、未明确要求写库的纠错，不调用本工具。
2. 写库指令必须明确包含“添加知识库：”“纠正知识库：”或“更新知识库：”。
3. 缺少正文授权 key、缺少标准答案、疑似重复时，不写入。
4. 写入成功后，只向用户说明新增条目 id、建议回测 query 和是否刷新本地索引；不要暴露环境变量、文件路径或 key。
